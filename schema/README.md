# Warp Commerce Model — Canonical Schema (v1.0.0)

This directory is the **language-neutral source of truth** for the Warp Commerce
Model. Every language binding is generated from / validated against these files,
so the bindings agree *by construction* rather than by hand-kept parallel edits.

```
schema/
├── VERSION                     1.0.0
├── structure/*.schema.json     the data shapes (types, discriminated unions, enums)
└── behavior/
    ├── transitions.json        the legal state-machine edges (Invariant 2 + 4)
    └── invariants.json         the six invariants + money_breakdown_sum + loyalty rules
```

Consumers:

- **TypeScript** — [`@warp-lang/commerce-types`](../packages/commerce-types) (npm).
- **Python** — [`warp-commerce-types`](../packages/commerce-types-py) (PyPI),
  generated by `packages/commerce-types-py/scripts/generate_from_schema.py`.

The Python package reads `behavior/transitions.json` and `behavior/invariants.json`
**at runtime** — the same files the conformance suite checks the TS package against —
so a change to a transition edge or an invariant message propagates to both languages
from one edit.

> Versioning note: this schema is `1.0.0`, the first consolidated, language-neutral
> cut. It is *ahead* of the TS package's `0.3.x` line by the additions a binding can
> adopt independently (e.g. `MoneyBreakdown`). Reconciliation brings every binding
> onto this version; the conformance suite is the gate.

## `structure/*.schema.json` grammar

Each file is `{ "$schema", "module", "doc", "definitions": { <Name>: <Def> } }`.
Definitions are resolved in **one global namespace** across all structure files,
so a type in one module may reference a type in another.

A `<Def>` is tagged by its `def` field:

| `def`     | shape | emits (Python) |
|-----------|-------|----------------|
| `newtype` | `{ "base": "string" }` | `Name = str` (branded id; brand is documentation only) |
| `enum`    | `{ "values": [...] }` | `Name = Literal[...]` |
| `alias`   | `{ "of": <TE> }` | `Name = <TE>` |
| `object`  | `{ "fields": { <f>: <Field> }, "validator"? }` | `class Name(BaseModel)` |
| `union`   | `{ "discriminator": "type"\|"kind"\|null, "variants": [...] }` | discriminated `Annotated[Union[...], Field(discriminator=...)]` or a plain `Union` |

A **variant** of a `union` is either `{ "tag": "Foo", "fields"?: {...} }` — which
generates a member class `NameFoo` with the discriminator pinned to `Literal["Foo"]` —
or `{ "ref": "OtherType" }` to include an already-defined object as a member.

A **`<Field>`** is either a type-expression `<TE>` (required, no default) or
`{ "t": <TE>, "optional"?: bool, "default"?: "list"|<value>, "doc"? }`.
`optional: true` ⇒ `Optional[...] = None`; `default: "list"` ⇒ `Field(default_factory=list)`.

A **`<TE>`** (type-expression) is one of:

- a string: a primitive (`string`, `number`, `integer`, `boolean`, `datetime`, `any`)
  or a reference to a definition by name (`"Money"`, `"PartyID"`, …);
- `{ "array": <TE> }` → `List[...]`;
- `{ "optional": <TE> }` → `Optional[...]`;
- `{ "literal": <value> }` → `Literal[value]`;
- `{ "enum": [<value>, …] }` → `Literal[...]`;
- `{ "ref": "Name" }` → explicit reference;
- `{ "union": [<TE>, …], "discriminator"? }` → `Union[...]` (discriminated if given).

Field names that are Python keywords (e.g. `from`) are emitted with a trailing
underscore and a Pydantic `alias` to the original name; models set
`populate_by_name=True` so both spellings construct.

## `behavior/transitions.json`

Per state machine (`commitment`, `intent`, `fulfillment`): an `edges` map from each
state type to the list of state types it may advance to. Any pair not listed is
rejected. `fulfillment.special_cases` carries the one payload-gated edge
(`Failed → Planned` only when `recoverable`). `history` declares the append-only /
monotonic-timestamp rule.

## `behavior/invariants.json`

`invariants` gives the `name`/`description`/`fix` for `I-1`…`I-6` (the violation
messages the checkers emit). `money_breakdown_sum` is the structural rule for
`MoneyBreakdown` (components sum to total, minor-unit tolerance, discounts negative,
single currency). `loyalty_liability` is Invariant 1's fourth clause.
